# 🧠 Dialogue Text Summarizer

A fine-tuned **T5-small** transformer model served via a **FastAPI** backend with a clean, responsive web interface — designed to generate concise summaries from multi-turn dialogues and long-form text.

---

## 📸 Preview

> Paste or type any dialogue or passage into the input field and receive an AI-generated summary in seconds.

```
Input  → Multi-turn conversation or long-form text
Output → Clean, concise summary
```

---

## 🚀 Features

- ✅ Fine-tuned `t5-small` model on the **SAMSum dialogue dataset**
- ✅ RESTful API built with **FastAPI**
- ✅ Lightweight frontend with **HTML, CSS & JavaScript** (no frameworks)
- ✅ Automatic device selection: **CUDA → MPS (Apple Silicon) → CPU**
- ✅ Text preprocessing pipeline (whitespace, HTML tag, newline cleanup)
- ✅ Beam search decoding (`num_beams=4`) for higher quality summaries
- ✅ Single-page app with async fetch — no page reloads

## ⚙️ Installation & Setup

### Prerequisites

- Python 3.9+
- pip

### 1. Clone the repository

```bash
git clone https://github.com/YOUR_USERNAME/dialogue-text-summarizer.git
cd dialogue-text-summarizer
```

### 2. Create and activate a virtual environment

```bash
python -m venv .venv

# On Windows
.venv\Scripts\activate

# On macOS / Linux
source .venv/bin/activate
```

### 3. Install dependencies

```bash
pip install -r requirements.txt
```

### 4. Add the model weights

The fine-tuned model is not stored in this repository due to file size. You have two options:

**Option A — Train it yourself** (see [Training](#model-training) section below)

**Option B — Download pre-trained weights**

Place the model files inside `TEXTSUMMARIZERAPP/saved_summary_model/`:
```
config.json
generation_config.json
model.safetensors
tokenizer_config.json
tokenizer.json
```

### 5. Run the application

```bash
cd TEXTSUMMARIZERAPP
uvicorn app:app --reload
```

Open your browser at: [http://127.0.0.1:8000](http://127.0.0.1:8000)

## 🔌 API Reference

### `GET /`

Returns the main HTML web interface.

---

### `POST /summarize/`

Summarizes the provided dialogue or text.

**Request Body:**
```json
{
  "dialogue": "Your text or dialogue here..."
}
```

**Response:**
```json
{
  "summary": "A concise summary of the provided content."
}
```

**Example with cURL:**
```bash
curl -X POST "http://127.0.0.1:8000/summarize/" \
     -H "Content-Type: application/json" \
     -d '{"dialogue": "Alice: Hey, are we still on for lunch? Bob: Yes, see you at noon!"}'
```

**Interactive API Docs** (auto-generated by FastAPI):
- Swagger UI → [http://127.0.0.1:8000/docs](http://127.0.0.1:8000/docs)
- ReDoc → [http://127.0.0.1:8000/redoc](http://127.0.0.1:8000/redoc)

## 🏋️ Model Training

The model was fine-tuned on the [SAMSum dataset](https://huggingface.co/datasets/Samsung/samsum) using the HuggingFace `Trainer` API.

### Training Configuration

| Parameter | Value |
|---|---|
| Base model | `t5-small` |
| Epochs | 6 |
| Train batch size | 8 |
| Eval batch size | 8 |
| Weight decay | 0.01 |
| Warmup steps | 500 |
| Max input length | 512 tokens |
| Max target length | 150 tokens |
| Evaluation strategy | Per epoch |
| Checkpoint saving | Per epoch |

### Run the training notebook

1. Download the SAMSum dataset CSVs and place them in the project root
2. Open `text_summarizer.ipynb` in Jupyter
3. Run all cells — the fine-tuned model will be saved to `./saved_summary_model/`

```bash
jupyter notebook text_summarizer.ipynb
```

### Data preprocessing

The training pipeline applies the following cleaning steps to both dialogues and summaries:

- Normalize `\r\n` line endings to spaces
- Collapse multiple whitespace characters
- Strip HTML tags (e.g. `<p>`, `<br>`)
- Trim and lowercase the text
